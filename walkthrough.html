Algumas Definições

O que é SQL?

SQL, que significa Structured Query Language (Linguagem de Consulta Estruturada), é uma maneira de interagir com bancos de dados relacionais e tabelas de uma forma que permite a nós, humanos, obter informações específicas e significativas.

Espere, o que é um banco de dados relacional?

Não há uma definição única para a palavra banco de dados. Em geral, bancos de dados são sistemas para gerenciar informações. Bancos de dados podem ter quantidades variadas de estrutura imposta aos dados. Quando os dados são mais estruturados, isso pode ajudar as pessoas e os computadores a trabalharem com os dados de maneira mais eficiente.

Bancos de dados relacionais são provavelmente o tipo mais conhecido de banco de dados. No seu cerne, bancos de dados relacionais são compostos por tabelas, que são muito parecidas com planilhas. Cada coluna na tabela tem um nome e um tipo de dado (texto, número, etc.), e cada linha da tabela é uma instância específica do que a tabela está "sobre". A parte "relacional" vem com regras específicas sobre como conectar dados entre diferentes tabelas.

O que é um ERD?

ERD, que significa Diagrama de Relacionamento de Entidades, é uma representação visual dos relacionamentos entre todas as tabelas relevantes dentro de um banco de dados. Você pode encontrar o ERD do nosso banco de dados de Mistério de Assassinato em SQL abaixo. O diagrama mostra que cada tabela tem um nome (parte superior da caixa, em negrito), uma lista de nomes de colunas (à esquerda) e seus tipos de dados correspondentes (à direita, em letras maiúsculas). Também há alguns ícones de chave dourada, ícones de seta azul e setas cinzas no ERD. Uma chave dourada indica que a coluna é a chave primária da tabela correspondente, e uma seta azul indica que a coluna é a chave estrangeira da tabela correspondente.

Chave Primária:
um identificador único para cada linha em uma tabela.
Chave Estrangeira:
usada para referenciar dados em uma tabela para os da outra tabela.
Se duas tabelas estão relacionadas, as colunas correspondentes, ou seja, os identificadores comuns das duas tabelas, são conectadas por uma seta cinza no diagrama.

O que é uma consulta?

Se você fosse olhar para os dados neste banco de dados, veria que as tabelas são enormes! Há tantos pontos de dados; simplesmente não é possível passar pelas tabelas linha por linha para encontrar as informações que precisamos. O que devemos fazer?

É aí que entram as consultas. Consultas são declarações que construímos para obter dados do banco de dados. Consultas lêem como inglês natural (na maior parte). Vamos tentar algumas consultas contra nosso banco de dados. Para cada uma das caixas abaixo, clique em "executar" para "executar" a consulta na caixa. Você pode editar as consultas aqui mesmo na página para explorar mais. (Observe que os comandos SQL não diferenciam maiúsculas de minúsculas, mas é convencional capitalizá-los para legibilidade. Você também pode usar novas linhas e espaços em branco como quiser para formatar o comando para legibilidade. A maioria dos sistemas de banco de dados exige que você termine uma consulta com um ponto e vírgula (';'), embora o sistema para executá-las nesta página da web seja mais tolerante.)
