Algumas Definições

O que é SQL?

SQL, que significa Structured Query Language (Linguagem de Consulta Estruturada), é uma forma de interagir com bancos de dados relacionais e tabelas de forma que nos permita extrair informações específicas e significativas.

O que é um banco de dados relacional?

Não há uma única definição para a palavra banco de dados. Em geral, bancos de dados são sistemas para gerenciar informações. Eles podem impor diferentes níveis de estrutura sobre os dados. Quando os dados são mais estruturados, isso pode ajudar as pessoas e os computadores a trabalhar com os dados de forma mais eficiente.

Bancos de dados relacionais são provavelmente o tipo mais conhecido de banco de dados. No seu núcleo, os bancos de dados relacionais são compostos por tabelas, que são muito parecidas com planilhas. Cada coluna na tabela tem um nome e um tipo de dado (texto, número, etc.), e cada linha na tabela é uma instância específica do que a tabela "representa". A parte "relacional" envolve regras específicas sobre como conectar dados entre diferentes tabelas.

O que é um ERD?

ERD, que significa Entity Relationship Diagram (Diagrama de Relacionamento de Entidades), é uma representação visual dos relacionamentos entre todas as tabelas relevantes dentro de um banco de dados. Você pode encontrar o ERD do nosso banco de dados do Mistério de Assassinato em SQL abaixo. O diagrama mostra que cada tabela tem um nome (na parte superior da caixa, em negrito), uma lista de nomes de colunas (à esquerda) e seus tipos de dados correspondentes (à direita, em letras maiúsculas). Há também alguns ícones de chave dourada, ícones de seta azul e setas cinzas no ERD. Uma chave dourada indica que a coluna é a ​chave primária​ da tabela correspondente, e uma seta azul indica que a coluna é a ​chave estrangeira​ da tabela correspondente.

Chave Primária​:
um identificador único para cada linha em uma tabela.
Chave Estrangeira​:
usada para referenciar dados em uma tabela a dados em outra tabela.
Se duas tabelas estiverem relacionadas, as colunas correspondentes, ou seja, os identificadores comuns das duas tabelas, serão conectadas por uma seta cinza no diagrama.

Quais elementos uma consulta SQL possui?

Uma consulta SQL pode conter:

  • Palavras-chave SQL (como o ​SELECT​ e ​FROM​ acima)
  • Nomes de colunas (como a coluna ​name​ acima)
  • Nomes de tabelas (como a tabela ​person​ acima)
  • Caracteres curinga (como ​%)
  • Funções
  • Critérios de filtragem específicos
  • Etc

Palavras-chave SQL

As palavras-chave SQL são usadas para especificar ações em suas consultas. As palavras-chave SQL não diferenciam maiúsculas de minúsculas, mas sugerimos usar letras maiúsculas para as palavras-chave SQL, para que você possa facilmente diferenciá-las do restante da consulta. Algumas palavras-chave frequentemente usadas são:

SELECT

SELECT​ nos permite obter dados para colunas específicas do banco de dados:

  • *​ (asterisco): é usado após ​SELECT​ para obter todas as colunas da tabela;
  • column_name(s): para selecionar colunas específicas, coloque os nomes das colunas após SELECT​ e use vírgulas para separá-las.
FROM

FROM​ nos permite especificar quais tabela(s) nos interessam; para selecionar várias tabelas, liste os nomes das tabelas e use vírgulas para separá-los. (Mas até você aprender a palavra-chave JOIN, pode ficar surpreso com o que acontece. Isso virá mais tarde.)

WHERE

A cláusula ​WHERE​ em uma consulta é usada para filtrar resultados por critérios específicos.

Curingas e outras funções para correspondências parciais

Às vezes você só sabe parte da informação de que precisa. O SQL pode lidar com isso. Símbolos especiais que representam caracteres desconhecidos são chamados de "curingas", e o SQL suporta dois. O mais comum é o curinga %.

Quando você coloca um curinga % em uma string de consulta, o sistema SQL retornará resultados que correspondem exatamente ao resto da string e têm qualquer coisa (ou nada) onde o curinga está. Por exemplo, 'Ca%a' corresponde a Canada e California.

O outro curinga, menos comumente usado, é _. Este significa 'corresponder ao restante do texto, desde que haja exatamente um caractere exatamente na posição do _, não importa qual seja. Assim, 'B_b' corresponderia a 'Bob' e 'Bub', mas não a 'Babe' ou 'Bb'.

Importante: Ao usar curingas, você não usa o símbolo =; em vez disso, você usa LIKE.

O SQL também suporta comparações numéricas como < (menor que) e > (maior que). Você também pode usar as palavras-chave BETWEEN e AND -- e todas elas funcionam com palavras, assim como com números.
