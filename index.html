Mistério do Assassinato em SQL

Você consegue descobrir quem é o assassino?

Houve um assassinato em SQL City! O Mistério do Assassinato em SQL foi criado para ser tanto uma lição autodirigida para aprender conceitos e comandos SQL quanto um jogo divertido para usuários experientes em SQL resolverem um crime intrigante.

Novato em SQL?

Este exercício é mais uma forma de praticar habilidades em SQL do que um tutorial completo. Se você nunca usou SQL, tente o tutorial. Se você realmente quer aprender muito sobre SQL, pode preferir um tutorial completo como Select Star SQL.

Se você já está confortável com SQL, pode começar a investigar!

Detetives experientes em SQL, comecem aqui

Um crime ocorreu e o detetive precisa da sua ajuda. O detetive lhe deu o relatório da cena do crime, mas você acabou perdendo. Você se lembra vagamente de que o crime foi um assassinato ocorrido em 15 de janeiro de 2018 e que aconteceu em SQL City. Comece recuperando o relatório da cena do crime do banco de dados do departamento de polícia.

Explorando a Estrutura do Banco de Dados

Usuários experientes em SQL podem frequentemente usar consultas de banco de dados para inferir a estrutura de um banco de dados. Mas cada sistema de banco de dados tem diferentes maneiras de gerenciar essas informações. O Mistério do Assassinato em SQL foi construído usando SQLite. Use este comando SQL para encontrar as tabelas no banco de dados do Mistério do Assassinato.

Além de conhecer os nomes das tabelas, você precisa saber como cada tabela é estruturada. A maneira como isso funciona também depende da tecnologia de banco de dados que você usa. Aqui está como fazer isso com SQLite.
